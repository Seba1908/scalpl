Please remove the scalpl project from Quicklisp's default dist #1917

adlai changed the title remove scalplPlease remove the scalpl project from Quicklisp's default distOct 16, 2020

There should be no dependents, and although github does report a daily clone rate that is somewhere within the margin of measurement error, I don't quite care whether anyone has built upon the undocumented internals.

The following quote is included in case any others would be affected by the removal of this project; if you wish to discuss the issue, please do so, although in a venue other than the one from which the quote was taken!
